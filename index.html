Spring Boot快速搭建应用实现登录注册

发表于 2017-05-17

Spring Boot能够快速构建项目，很容易的创建一个独立运行的项目，可以不用或者只需要很少的Spring配置。这篇文章就记录下用Spring Boot快速搭建应用实现登陆注册功能过程中的一些知识点和逻辑。

Spring Boot快速搭建

在 http://start.spring.io 网站里，我们可以利用Maven作为构建方式来快速搭建我们的项目，在添加依赖里选择Web Velocity ，然后点击下载，解压后用IDEA打开pom.xml文件整个项目就加载起来了，如果项目以后需要导入哪个第三方jar包时可以直接在pom.xml文件中添加相关jar包依赖，Maven会根据dependency中的依赖配置，通过互联网在Maven中心库下载依赖包到本地。

之前选择的Web会在Maven里依赖 spring-boot-starter-web，表示对Web项目开发的支持，为我们提供了嵌入的Tomcat以及Spring MVC的依赖；Velocity 依赖 spring-boot-starter-velocity表示对Velocity模板引擎的支持。

结构

  • controller：控制器(SpringMVC注解@Controller的类)，引用srivce层来控制业务流程。
  • dao：数据库访问层，Data Access Objiect(数据访问对象)， 数据库访问对象的第一个面向对象的接口，操作数据库 （增删改查）。
  • service：业务层， 建立在Dao数据库访问层之上，引用对应的Dao层数据库操作。

  • model：数据模型，对应数据库表的实体类和数据库表的字段是基本对应的。

  • view：视图页面 包含jsp,Velocity,Thymeleaf等。

  • util：工具类。
  • static：可以用来存放静态资源。
  • templates：用来存放默认的模板配置路径。

Python爬取v2ex上收藏的主题

发表于 2017-04-15

之前上v2ex发现自己收藏的帖子已经有500个了，于是想用爬虫将这些主题的标题和链接保存到本地，防止以后忘了或者一些原因登录不上。用到了request和Beautiful Soup两个模块，实现很简单，比较新手。

模拟登录

当我们登录一个网站时服务器首先要验证用户名和密码，然后服务器会下发一个token(或者是sessionid)关联userID表明是我们身份,然后存放到cookie中，下次在一定时间内登录就不需要再输入密码了。所以我们模拟登录时可以提交用户名和密码（但很可能被加密了，登录不上），也可以带cookie提交。

一开始我模拟登录v2ex是想在网页上登陆后将From Data里的数据在代码里原封不动写上POST上去的，但试了几次不行，后来看了别人的代码发现要带上Headers 里面的Refer, Origin，于是登录就参考了别人的代码，也理解下思路。v2ex登录需要提交用户名,密码和once ，once和用户名、密码的key都是随机生成的，登录时可以在源代码里用正则表达式或者用Beautiful Soup找到。

正则表达式学习

发表于 2017-03-22

最近在看一些防止xss攻击的文章，发现简单的可以用正则表达式来替换一些符号，想起来第一次接触正则表达式是之前学习爬虫的时候，当时是爬知乎里的一些图片，用的方法是在源代码里找到每张图片代码前后相同字符的部分然后用正则表达式进行匹配这些部分，但之前是跟着视频做的，并没有太了解正则表达式，这次就系统的学习下正则表达式，主要总结正则表达式的一些基本的知识点。

正则表达式是用于匹配字符串中字符组合的模式。一个正则表达式模式可以是由简单的字符所构成的，也可以是简单和特殊字符的组合。

字符类

  • . ：匹配除换行符以外的任意字符。

  • \d：匹配数字，等同于[0-9]。

    例如 /\d/ 找到 “my number is 1316” 中的’1316’。

  • \w：匹配字母或数字或下划线，等同于[A-Za-z0-9_]。

  • \s：匹配任意的空白符。

  • \b：匹配一个单词的边界，开始或结束。

    例如/\ba/ 找到“cake apple”中apple单词里的‘a’；/le\b/找到 “telephone apple” 中apple单词里的’le’，

  • ^：匹配输入的开始。

过滤器Filter

发表于 2017-03-16

在学生信息管理的例子中时，主页面应该是需要用户登录后才能查看的，但是用户可以在未登陆的时候在浏览器地址输入url直接访问，这里可以用过滤器来对用户请求进行统一认证即如果没有登录(session中获取不到用户信息)就全部转发到登录界面；并且可以同时在过滤器中设置编码转换，这样就不需要当请求里含有中文时都需要在提交到的Servlet中设置请求编码和响应编码问题。

另外之前看到一个是视频讲到要对所有用户能输入的地方加以小心，例如我在测试时input里输入<script>alert("hello")</script> 每次刷新的话都会弹出；如果对用户输入的类型不加以识别就会很不安全，上网搜了下发现可以使用过滤器对每个 post 请求的参数过滤一些关键字，替换成安全的；当然防范攻击的方式有很多不止使用过滤器，还可以在前端来进行防范。上次在论坛里也看到一个帖子也是关于前端和后端防范攻击讨论，这个以后需要好好了解下，这次就先总结下过滤器的一些知识吧。

Filter

过滤器主要用于对用户请求进行预处理，也可以对HttpServletResponse进行后处理，是个典型的处理链。使用Filter的完整流程是：Filter对用户请求进行预处理，接着将请求交给Servlet进行处理并生成响应，最后Filter再对服务器响应进行后处理。
