最近在完成一个简单的学生信息管理的例子，使用Jsp和Servlet实现了基本功能，数据层使用JDBC和MySQL，这次就记下一些用到的JDBC和MySQL的知识。

JDBC简介

JDBC(java database connectivity)为Java数据库连接，是一种用于执行SQL语句的Java API，JDBC就是用Java语言来向各种关系型数据库发送sql语句从而来操作数据库。

使用JDBC代码查询数据库的信息流程是我们需要创建一个数据库连接，然后再创建一个语句对象，接着进行查询；在最后我们需要关闭数据库连接、语句和结果集(关闭资源时采用倒关的手法)。

基本的增、删、改、查语句

  • 插入数据: insert into 表名 (列名1，列名2，列名3) values (列值1，列值2，列值3) 一次插入就是一行

JDBC对数据库进行CRUD操作

Statement

用于执行SQL语句的工具接口，可以执行SQL查询，返回查询到的结果集。

常用Statement方法：

  1. boolean excute(String sql) throws SQLException:执行任何SQL语句，如果执行后第一个结果为ResultSet对象则返回true，如果为受影响的行数或没有结果返回false。
  2. ResultSet excuteQuery(String sql) throws SQLException:只能执行select语句，执行后返回查询结果对应的ResultSet对象。
  3. int excuteUpdate(String sql) throws SQLException:执行DML语句，如insert/update/delete语句时返回更新的行数；执行DDL语句返回0。

PreparedStatement

PrepaerdStatement是Statement的子接口，它允许数据库预编译SQL语句，以后每次只需改变SQL命令参数，避免数据库每次都需要编译SQL语句。使用PrepaerdStatement执行SQL语句，无须再传入SQL语句，只要为SQL语句传入参数。PrepaerdStatement对象比Statement对象效率更高，而且也可以防止SQL注入，所以一般用PrepaerdStatement。

PrepaerdStatement同样有excuteUpdate()，excuteQuery()和excute()三个方法，只是三个方法不需要传入SQL字符串，因为PrepaerdStatement对象预编译SQL命令，只要为这些命令传入参数。

ResultSet

Statement执行SQL语句时返回ResultSet结果集。

常用ResultSet方法移动记录指针：

  1. void close():释放ResultSet对象。
  2. boolean first():将ResultSet的记录指针定位到首行。如果移动后的记录指针指向一条有效记录则返回true。
  3. boolean previous():将ResultSet的记录指针定位到上一行。如果移动后的记录指针指向一条有效记录则返回true。
  4. boolean next():将ResultSet的记录指针定位到下一行。如果移动后的记录指针指向一条有效记录则返回true。

当把记录指针移动到指定行之后，ResultSet可以通过getXxx(int columnIndex)方法来获取当前行指定列的值。

  1. getInt():获得在数据库里时int类型的对象。
  2. getString():获得在数据库里时varchar,char类型的对象。
  3. getFloat():获得在数据库里时float类型的对象。
  4. getDate():获得在数据库里时date类型的对象。
  5. getBoolean():获得在数据库里时boolean类型的对象。
