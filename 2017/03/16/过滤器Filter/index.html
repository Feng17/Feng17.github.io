过滤器Filter

发表于 2017-03-16

之前在电脑上测试学生信息管理的代码时，发现当用户没有登录时也能访问到其它页面，于是再重新看了下书之后用过滤器来对用户请求进行统一认证即如果没有登录(session中获取不到用户信息)就全部转发到登录界面；并且同时在过滤器中设置了编码转换，这样就不需要当每个POST请求含有中文时都需要在提交到的Servlet中设置请求编码和响应编码问题。另外之前我在测试时发现要对input输入类型进行限制，如果用户输入的不是学号或其它的而是一段代码例如 <script>alert("hello")</script> 这样每次刷新的话都会弹出；如果对用户输入的类型不加以识别就会很不安全，上网搜了下发现可以使用过滤器对每个 post 请求的参数过滤一些关键字，替换成安全的，当然防范攻击的方式有很多不止使用过滤器，还可以在前端来进行防范，这个以后再好好研究下，这次就先总结下过滤器的一些知识吧。

Filter

过滤器主要用于对用户请求进行预处理，也可以对HttpServletResponse进行后处理，是个典型的处理链。使用Filter的完整流程是：Filter对用户请求进行预处理，接着将请求交给Servlet进行处理并生成响应，最后Filter再对服务器响应进行后处理。

创建Filter类

1.创建一个类继承Filter接口，接口中定义了如下三个方法。

  • void init()：Filter的初始化，Filter在服务器启动时就创建，创建之后马上执行这个方法。用来初始化一些参数
  • void doFilter(req,resp,chain)：当向服务器请求的Servlet或jsp页面在过滤器的过滤范围内时就会执行这个方法。若方法体中没有chain.doFilter()操作，则表示当向服务器请求该过滤器过滤范围内的资源(如Servlet/JSP页面/html页面等)时，这些资源中的所有方法都不会执行(被过滤掉了);若方法体中有chain.doFilter()操作，表示不对过滤器过滤范围内的资源进行过滤。(即你请求的资源下的方法会执行)
  • void destroy()：在服务器关闭时对Filter进行销毁，在Filter销毁之前会执行这个方法，完成某些资源的回收。

配置Filter

配置Filter同样有两种方式 一是通过注解 通过@WebFilter( )修饰一个Filter类，用于对Filter进行配置。

二是在web.xml中通过配置文件配置

  • 过滤器通过 Web 部署描述符（web.xml）中的 XML 标签来声明，然后映射到你的应用程序的部署描述符中的 Servlet 名称或 URL 模式。当Web容器启动Web应用程序时，它会为你在部署描述符中声明的每一个过滤器创建一个实例。Filter的执行顺序与在web.xml配置文件中的配置顺序一致，一般把Filter配置在所有的Servlet之前。

过滤器的四种拦截方式

  • 拦截直接请求方式:REQUEST
  • 拦截请求转发方式:FORWARD
  • 拦截请求包含方式:INCLUDE
  • 拦截错误转发方式:ERROR

实现不同的拦截方式需要在中进行不同的配置:

  • <dispatvher>REQUEST</dispatvher>
  • <dispatvher>FORWORD</dispatvher>
  • <dispatvher>INCLUDE</dispatvher>
  • <dispatvher>ERROR</dispatvher>

若在web.xml配置文件中没有写出上面四个拦截配置时默认该过滤器只拦截请求。
